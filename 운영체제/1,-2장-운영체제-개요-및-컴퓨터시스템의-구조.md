## **운영체제란?**

컴퓨터 하드웨어 바로 위에 설치되어 사용자 및 다른 모든 소프트웨어와 하드웨어를 연결하는 소프트웨어 계층

![img](https://blog.kakaocdn.net/dn/dYzWtE/btrbwgSqIxJ/DCqOk6KOQBJUaR0y804P10/img.png)

좁은 의미의 운영체제 -> 커널

커널 : 운영체제의 핵심 부분으로 메모리에 상주하는 부분

넓은 의미의 운영체제

\- 커널 뿐 아니라 각종 주변 시스템 유틸리티를 포함한 개념(별도의 프로그램)



## **운영 체제의 목적**

가장 큰 목적★

컴퓨터 시스템의 자원을 효율적으로 관리

\- 프로세서, 기억장치,입출력 장치 등의 효율적 관리

① 사용자간의 형평성 있는 자원 분배

② 주어진 자원으로 최대한의 성능을 내도록(효율성)

\- 사용자 및 운영체제 자신의 보호

컴퓨터 시스템을 편리하게 사용할 수 있는 환경을 제공

\- 운영체제는 동시 사용자/ 프로그램 들이 각각 독자적 컴퓨터에서 수행되는 것 같은 환상을 제공한다.

\- 하드웨어를 다루는 복잡한 부분은 운영체제가 대신 수행해줍니다.



## **운영 체제의 분류**

① 동시 작업 가능 여부

② 사용자의 수

③ 처리 방식



#### **동시 작업 가능 여부**

단일 작업/다중 작업 으로 분류

대부분 다중 작업으로 분류됩니다.

단일 작업은 MS-DOs 프롬프트 상에서는 한 명령의 수행을 끝내기 전에 다른 명령을 수행시킬 수 없음.(설계가 쉬움)

현재 대부분 운영체제는 다중 작업으로 분류 됩니다.

#### **사용자의 수**

단일 사용자/ 다중 사용자

단일사용자:MS-DOS/MS Windows

다중 사용자:UNIX,NT server

#### **처리 방식**

일괄 처리(batch processing)

\- 작업 요청의 일정량을 모아서 한꺼번에 처리

\- 작업이 완전 종료될 때까지 기다려야함

시분할(time sharing)
\- 여러 작업을 수행할 때 컴퓨터 처리 능력을 일정한 시간 단위로 분할하여 사용 

\- 짧은 응답 시간을 가짐

\- 현대 대부분 운영체제가 시분할 처리 방식으로 분류 됩니다.

실시간(Realtime OS)

\- 정해진 시간 안에 어떠한 일이 반드시 종료됨이 보장되어야 하는 실시간시스템을 위한 OS

예) 미사일 제어, 반도체 장비

실시간 ->데드라인이 존재, 특수한 목적이 존재한다. 

시분할 ->일반적인 컴퓨터





## 운영 체제의 예

유닉스(UNIX)

\- 코드의 대부분을 C언어로 작성

\- 높은 이식성,높은 확장성

\- 다양한 버전(대표적Linux)

MS Winodows

\- MS사의 다중 작업용 **GUI 기반** 운영 체제

\- 초창기에는 불안정성이 존재하였습니다.(지금은 X)

\- 개인용 컴퓨터를 위한 운영체제

### CPU(항상 메모리와 일을 한다)

register:memory보다 더 빠르면서 저장을 하는 작은 공간

mode bit:CPU에서 실행되는 것이 운영체제인지 사용자 프로그램인지 구분해주는 것(보안을 목적으로)

Interrupt line: 주변기기가 CPU에게 어떤 사실을 알리는 일

만약 CPU와 Disk와 일을 할 때에는 **device controller**에게 시킨다.

**device controller(장치 제어기):각** 장치를 통제하는 일종의 작은 CPU->hardware





만약 무한루프의 사용자 프로그램의 경우를 대비해

**timer(시간을 걸어 두는 개념)를** 둔다 특정 프로그램이 CPU를 독점하는 것을 막기 위해

IoDevice은 운영체제를 통해서만 가능하다.



타이머

-정해진 시간이 흐른 뒤 운영체제에게 제어권이 넘어가도록 인터럽트를 발생시킴





DMA cotroller

CPU가 일을 할 때에 device controller가 너무 자주 부르게 되면 효율적이지 못하게 됩니다. 이러한 것을 방지하기 위한 것입니다.

DMA가 CPU가 하는 일들 중 localbuffer에 있는 내용을 메모리에 직접 복사까지 해주고 CPU에 인터럽트를 합니다. 그럼 인터럽트의 빈도수가 줄어들게 됩니다. 



memory controller

-메모리 접근이 DMA도 가능하고 CPU도 가능해서 교통정리 역할을 하는 것



사용자 프로그램은 어떻게 I/O를 하는가?

\- 시스템 콜 : 사용자 프로그램이 운영체제에게 I/O 요청, 커널 함수를 호출하는 경우

\- trap을 사용하여 인터럽트 벡터의 특정 위치로 이동

\- 올바른 I/O 요청인지 확인 후 I/O 수행

\- I/O 완료 시 제어권을 시스템 콜 다음 명령으로 수행



인터럽트: 인터럽트 당한 시점의 레지스터와 program controller를 save 한 후 CPU의 제어를 인터럽트 처리 루틴에 넘긴다.(타이머, I/O controller)

넓은 의미(하드웨어 +소프트웨어)

하드웨어(Interrupt)는 타이머, I/O controller를 뜻하고

소프트웨어(Trap)는

①Exception:프로그램이 오류를 범한 경우

②System call:프로그램이 커널 함수를 호출하는 경우
